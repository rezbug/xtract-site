Leia e entenda a estrutura do projeto

Escreva codigo usando boas praticas: 
- YAGNI 
- KISS
- DRY
- Modularização
- Serviços ao invés de hooks

Evite anti-padrões

Siga a documentação da biblioteca slash:
https://slash-docs.pages.dev/intro/conceitos-core/

Regras para Typescript:
- Nunca use o tipo any
- Sempre tipar corretamente todos os parametros de funções, objetos, variáveis e demais tipos de dados

Lint e Qualidade de código:
- Use biome para verificar problemas de lint e qualidade de codigo
- Documentação do biome:
https://biomejs.dev/pt-br/guides/getting-started/


Tecnologias usadas no projeto:
- Slash (biblioteca reativa)- referencia(https://github.com/rezbug/slash, https://slash-docs.pages.dev/intro/conceitos-core/)
- biome - referencia (https://biomejs.dev/pt-br/guides/getting-started/)
- bun - referencia (https://bun.com/docs)
- htm - referencia (https://github.com/developit/htm)


Plano de ação:
-Todo plano de ação ou lista de tarefa deverá ser registrado na pasta ~/.config/codex em algum arquivo {nome-do-arquivo}.md
-Toda tarefa registrada deverá ser atualizada sempre que uma etapa for concluida

Exemplo: 
- Progresso de tarefas
[] 1.0 - {descrição }
[x] 1.1 - {descrição }
[x] 1.2 - {descrição }
[ ] 1.3 - {descrição }


Observação importante: 
Após concluir cada etapa do plano de ação gere uma mensagem de commit e descreva usando conventional commits:
- Ação
- Objetivo
- Aquivos modificados 

Eu mesma farei o commit gere apenas a mensagem
